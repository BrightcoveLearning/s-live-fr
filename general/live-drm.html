DRM en direct

Dans cette rubrique, vous apprendrez comment activer la protection du contenu pour votre diffusion en direct.

Activation des DRM pour le contenu en direct

Remarque:

Par défaut, le DRM est appliqué aux sorties comme suit:

  • Votre compte Live doit d'abord être activé par le support Brightcove pour pouvoir utiliser la gestion des droits numériques.
  • Pour publier un événement prenant en charge la gestion des droits numériques, utilisez l'une des méthodes suivantes:
    • Un lecteur Brightcove v6.37.1 ou version ultérieure compatible DRM
    • SDK natif Brightcove pour iOS v6.5.0 ou version ultérieure

    • SDK natif Brightcove pour Android v6.11.0 ou version ultérieure

  • Si vous souhaitez remplacer les paramètres par défaut de DRM, vous pouvez le faire via l'API ou contacter le support Brightcove pour les modifier dans Live Platform.
    • FairPlay: appliqué aux sorties HLS
    • PlayReady: appliqué aux sorties DASH
    • Widevine: appliqué aux sorties HLS [1-1]

Remarques

  • [1-1] Widevine peut être appliqué aux sorties DASH au lieu de HLS. Pour que Widevine avec DASH soit la valeur par défaut, il faut un remplacement au niveau du compte. Contactez le support si vous souhaitez que ce remplacement soit appliqué à votre compte. À l'aide de l'API Live, vous pouvez également spécifier widevine:hls ou widevine:dash pour des tâches spécifiques, quelle que soit la valeur par défaut de votre compte. Notez que selon la façon dont votre compte/travail est configuré, la réponse de l'API (comme celle présentée ci-dessous) inclura les sorties widevine:hls ou widevine:dash, mais pas les deux.

Technologies de lecteur utilisées

Le lecteur Brightcove utilise différentes technologies de lecteur DRM avec différents navigateurs. Les technologies DRM utilisées avec Brightcove Player sont les suivantes:

  • FairPlay: Le système DRM d'Apple
  • Prêt pour le jeu: Le système DRM de Microsoft
  • Widevine: Le système DRM de Google

Le tableau suivant détaille la relation entre le navigateur (dernière version), le format et la technologie de lecteur utilisés dans Brightcove Player:

pour diffuser du contenu DRM
Ordinateur de bureau Chrome HLS avec Widevine HLS natif HLS
Chrome Mobile 1 HLS avec Widevine HLS natif HLS
Internet Explorer Aucune version d'IE n'est prise en charge par Brightcove
Edge DASH avec PlayReady Natif/EME MPEG-DASH
Safari HLS avec FairPlay HLS natif HLS
Firefox HLS avec Widevine HLS natif HLS

Widevine peut être appliqué aux sorties DASH au lieu de HLS. Pour que Widevine avec DASH soit la valeur par défaut, il faut un remplacement au niveau du compte. Contactez le support si vous souhaitez que ce remplacement soit appliqué à votre compte.

Pour activer la gestion des droits numériques pour votre contenu Live, procédez comme suit:

  1. Lors de la création d'un événement en direct, dans Options avancées > Chiffrement du contenu. Sélectionnez Digital Rights Management (DRM) dans la liste déroulante Chiffrement. Gestion des droits numériques (DRM): flux de contenu Premium sécurisé grâce aux technologies de gestion des droits numériques.
    • HLS + FairPlay
    • HLS (FMP4) +Widevine Modular
    Votre compte Live doit d'abord être activé par le support Brightcove pour pouvoir utiliser la gestion des droits numériques. Pour publier un événement compatible DRM, il est nécessaire de créer et d'utiliser un lecteur compatible DRM v6.37.1 ou version ultérieure.
  2. Une option supplémentaire d'expiration du jeton apparaît, vous permettant de sélectionner la durée en secondes avec un délai d'expiration minimum de 900 secondes et un maximum de 259 2000 secondes (le dernier étant égal à 30 jours).
  3. Lorsque l'événement est créé, dans le panneau d'informations de l'événement, vous pourrez voir les URL DRM pour Fairplay et Widevine.
  4. Pour publier un événement compatible DRM, cliquez sur Publier et intégrer dans le volet de navigation de gauche.
    • Dans la section Options du lecteur en direct, sélectionnez un lecteur compatible DRM. (Pour la lecteur DRM en direct, vous devez sélectionner ou créer un lecteur compatible DRM v6.37.1 ou version ultérieure).
    • Si vous n'avez pas de lecteur DRM disponible, créez-en un à l'aide du module Lecteurs.™ Dans les paramètres de lecteur, assurez-vous que l'option Activer DRM est sélectionnée. (La lecteur de Player for Live DRM doit être la version 6.37.1 ou supérieure).
  5. De retour dans le module Live, vous pourrez sélectionner votre lecteur compatible DRM et publier l'événement.
